# 搜索页面排版优化 - 实施计划

> **选择方案**: 方案 A - 顶部搜索区 + 结果区

## 问题分析

**当前布局顺序**（从上到下）：
1. 找到 X 个仓库
2. 📊 当前结果语言分布图
3. 🔗 关联推荐
4. 🔍 搜索框和筛选器
5. 搜索结果网格

**核心问题**：
- ❌ 搜索框在页面底部，用户体验差
- ❌ "关联推荐"在搜索结果上方，逻辑颠倒
- ❌ 语言分布图占用显眼位置
- ❌ 搜索和筛选器分散

---

## 设计方案：方案 A - 顶部搜索区 + 结果区

### 新布局结构

```
┌──────────────────────────────────────────────────────┐
│ 🔍 搜索和筛选区（顶部）                        │
│ ┌─────────────────────────────────────────────────────┐ │
│ │ [搜索框] [搜索按钮]                               │ │
│ ┌─────────────────────────────────────────────────────┐ │

└──────────────────────────────────────────────────────┘

```

## 实施步骤

### Step 1: 调整布局顺序

#### 1.1 移动搜索和筛选器到顶部

**当前代码位置**：第 52-151 行
**目标位置**：移到页面最顶部（第 2-6 行之后）

**操作**：
1. 剪切 `<!-- Search and Filters -->` 区块（第 52-151 行）
2. 粘贴到 `<div class="space-y-6">` 的最前面
3. 调整样式为卡片样式

#### 1.2 优化搜索框样式

**优化内容**：
- 添加卡片容器样式（白色/深灰色背景，圆角，阴影）
- 优化输入框和按钮样式
- 筛选器使用更紧凑的布局

---

### Step 2: 调整关联推荐位置

**当前代码位置**：第 22-50 行
**目标位置**：移到搜索结果网格之后

**操作**：
1. 删除当前位置的 "Related Recommendations" 区块
2. 在 Results Grid 后重新添加 "关联推荐" 区块
3. 更新标题为：`h3 class="text-xl font-bold text-gray-900 dark:text-white mt-8 mb-4"`

---

### Step 3: 删除或下移语言分布图

**当前代码位置**：第 8-20 行

**选项**：

### Step 4: 优化筛选器样式

**当前代码**：分散在下拉框和复选框
